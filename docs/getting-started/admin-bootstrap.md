# Admin Bootstrap

Quizz does not create a default admin account. The admin bootstrap script is a controlled setup tool for creating or updating the configured admin user after the application schema exists.

## Why Bootstrap Exists

Admin access is required to create categories, questions, and quizzes. A built-in default admin would create an unsafe credential path, so the project uses an explicit interactive bootstrap flow.

## Prerequisites

- `.env` exists and includes:
  - `QUIZZ_DEFAULT_ADMIN_EMAIL`
  - `QUIZZ_DEFAULT_ADMIN_FULL_NAME`
  - `POSTGRES_DB`
  - `POSTGRES_USER`
- Docker runtime can start `postgres` and `quizz`.
- `docker/secrets/postgres_password.txt` exists.

## Run

```bash
scripts/bootstrap-admin.sh
```

The script prompts for the admin password twice. The password is not passed as a command-line argument and is unset after hash generation. The password hash is generated by running the application jar in `hash-password` mode, which reads from standard input, writes the hash to standard output, and writes validation errors to standard error.

## Behavior

- Starts the required Compose services if needed.
- Waits for PostgreSQL readiness.
- Waits for Flyway to create the `users` table.
- Upserts the admin user with `role = ADMIN` and `enabled = true`.
- Uses `scripts/sql/upsert-admin.sql` as the controlled direct database exception.

## Verify

1. Open `http://localhost:<QUIZZ_HTTP_PORT>/login`.
2. Sign in with the configured admin email and the password entered at the prompt.
3. Confirm the admin dashboard is accessible at `/admin`.

## Troubleshooting

- Missing `.env`: copy `.env.example` to `.env` and review values.
- Missing secret file: create `docker/secrets/postgres_password.txt`.
- Password rejected: use at least 12 characters and confirm the same value.
- Login fails after bootstrap: rerun the script and enter the intended password interactively.
